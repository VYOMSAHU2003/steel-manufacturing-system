## 📋 Overview

The **Steel Manufacturing Plant Management System** is a full-stack enterprise solution designed to streamline and optimize steel manufacturing operations. Built with Python and Streamlit, it provides real-time monitoring, comprehensive analytics, and seamless workflow management across all manufacturing processes.

### 🎯 Key Highlights
- **Real-time Operations Monitoring**
- **Role-based Access Control**
- **Comprehensive Analytics Dashboard**
- **Enterprise-grade Database Integration**
- **Modern, Responsive UI/UX**

### ⚡ Quick Start

```bash

git clone https://github.com/VYOMSAHU2003/steel-manufacturing-system.git
cd steel-manufacturing-system

# 2️⃣ Create virtual environment
python -m venv venv

# 3️⃣ Activate virtual environment
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 4️⃣ Install dependencies
pip install -r requirements.txt

python scripts/init_db.py

# 6️⃣ Launch application
streamlit run app.py
```

### 🎉 Success!
Open your browser and navigate to: **http://localhost:8501**

## Database Schema

### Core Tables

1. **users** - User accounts and role management
2. **raw_materials** - Raw material inventory
3. **production_orders** - Production order tracking
4. **inventory_logs** - Material transaction history
5. **quality_inspections** - Quality test results
6. **shipments** - Shipment and logistics tracking

Each table includes audit columns (created_at, updated_at) for tracking changes.

## Configuration

### Oracle Database Connection

To connect to an Oracle database instead of SQLite:

1. Update `.env` with your Oracle credentials:
```
ORACLE_HOST=your_host
ORACLE_PORT=1521
ORACLE_SERVICE_NAME=your_sid
ORACLE_USER=your_user
ORACLE_PASSWORD=your_password
```

2. The application will automatically detect and use the Oracle connection

### Environment Variables

- `ORACLE_HOST` - Oracle database host
- `ORACLE_PORT` - Oracle database port (default: 1521)
- `ORACLE_SERVICE_NAME` - Oracle service name/SID
- `ORACLE_USER` - Oracle database user
- `ORACLE_PASSWORD` - Oracle database password
- `DEBUG_MODE` - Enable debug logging (default: false)

## API Endpoints

The system uses SQLAlchemy ORM for database operations. All database interactions go through the `SessionLocal` connection manager configured in `config/database.py`.

## Security Features

- **Password Hashing:** bcrypt for secure password storage
- **Role-Based Access Control:** Granular permissions per user role
- **Session Management:** Secure session state management via Streamlit
- **Input Validation:** Data validation on all forms
- **SQL Injection Prevention:** SQLAlchemy parameterized queries

## Troubleshooting

### Database Connection Issues
- Verify Oracle credentials in `.env`
- Ensure Oracle service is running
- Check network connectivity to database host
- The system will fall back to SQLite if Oracle fails

### Missing Tables
- Run `python scripts/init_db.py` to create tables
- Check database user has CREATE TABLE permissions

### Import Errors
- Ensure virtual environment is activated
- Run `pip install -r requirements.txt` again
- Check Python version is 3.8+

### Login Issues
- Use default credentials from table above
- Check if database was initialized with `init_db.py`
- Clear browser cache and try again

## Performance Optimization

- Database connection pooling enabled
- Lazy-loading of relationships
- Query result caching where applicable
- Indexed foreign keys for faster lookups
